Who thinks publishing
Linked Data on the Web
is a good idea?

Who publishes their
own Linked Data?

Who enables people to
query their Linked Data?

My open-source pipeline
makes Linked Data on websites queryable.

This pipeline unlocks
the value of your data
for client-side applications.

Piecing the puzzle

  • Linked Data on personal websites
  • Making our Linked Data queryable
  • Open questions for data publishers

My personal website contains metadata
about my research and publications.

This includes metadata about:

  • 40 blog posts
  • 200 publications
  • 2 Linked Research articles

I publish RDF the “Linked Data” way
through Turtle and RDFa.

  • FOAF profile in Turtle
    • partly manual, mostly autogenerated
    • ≥6,400 RDF triples about myself, people, publications
  • templated HTML pages with RDFa
    • autogenerated
    • ≥13,000 RDF triples about publications, blog posts, articles
    • (±50 triples per page)

I publish my own Linked Data because
we need to practice what we preach.

  • How can we convince people
    they should publish 5-star Linked Data
    if we continue making up excuses
    for not doing it ourselves?
  • It’s the home-cooking version of
    Semantic Web Dog Food and ScholarlyData.org.

I publish my own Linked Data because
others already publish it—wrongly.

I struggle to keep up with the incompleteness, inaccuracies, duplicates, and wrong entries of:

  • Mendeley
  • ResearchGate
  • Academia
  • ORCID
  • Web of Science
  • Springer’s SciGraph
  • Semantic Web Dog Food
  • ScholarlyData.org
  • my own institution’s repository
